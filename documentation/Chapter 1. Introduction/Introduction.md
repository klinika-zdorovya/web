# **1. Введение и обзор проекта**

## **1.1. Назначение сайта**

Сайт "Клиника здоровья" представляет собой официальный информационный ресурс медицинского учреждения. Основные цели сайта:

1. **Информирование пациентов** о спектре медицинских услуг, специалистах клиники и условиях предоставления помощи
2. **Повышение доверия** к клинике через публикацию документов, истории создания, манифеста и отзывов
3. **Привлечение новых пациентов** за счёт подробного описания направлений деятельности и демонстрации экспертизы через публикации, новости и полезные советы
4. **Упрощение доступа к информации** через структурированные разделы с ценами, контактами и ответами на частые вопросы

Сайт выполняет функции визитной карточки, информационного портала и инструмента маркетинга одновременно.

## **1.2. Целевая аудитория**

### **Основные категории пользователей:**

1. **Пациенты и их родственники**
    - Ищущие информацию о конкретных медицинских услугах (реабилитация, ЛФК и др.)
    - Желающие ознакомиться с ценами и документами клиники
    - Ищущие контактные данные и график работы

2. **Потенциальные партнёры**
    - Организаторы спортивных мероприятий (для направления "Обеспечение спортивных мероприятий")
    - Поставщики ортопедической продукции
    - Другие медицинские учреждения для сотрудничества

3. **Журналисты и исследователи**
    - Ищущие экспертное мнение или материалы для публикаций

4. **Сотрудники клиники** (как редакторы контента)
    - Обновляющие информацию через GitHub

## **1.3. Технологический стек**

Проект использует современные веб-технологии, обеспечивающие высокую производительность, удобство разработки и простоту поддержки:

| Компонент | Технология | Назначение | Версия  |
|-----------|------------|------------|---------|
| **Фреймворк** | Nuxt | Vue-фреймворк для создания универсальных веб-приложений с серверным рендерингом (SSR) и статической генерацией (SSG) | 3.x     |
| **Управление контентом** | Nuxt Content | Git-ориентированный headless CMS, позволяющий хранить контент в файлах Markdown, YAML, JSON | 3.x     |
| **Стилизация** | Tailwind CSS | Utility-first CSS фреймворк для быстрой и согласованной стилизации без написания кастомного CSS | 6.x     |
| **Язык программирования** | JavaScript | Основной язык разработки логики приложения | ES2020+ |
| **Система контроля версий** | Git | Отслеживание изменений в коде и контенте | 2.x+    |
| **Хостинг кода** | GitHub | Хранение репозитория, управление версиями, совместная работа | -       |
| **Деплой** | GitHub Actions + FTP | Автоматизированный процесс развёртывания на хостинг после каждого коммита | -       |
| **Контент-форматы** | Markdown (.md), YAML (.yml/.yaml), JSON (.json) | Форматы хранения структурированного и неструктурированного контента | -       |

### **Ключевые преимущества Nuxt Content 3:**

- **Улучшенная производительность**: Более быстрая загрузка контента за счет оптимизированной работы с файловой системой
- **Улучшенный API**: Упрощенный и более мощный API для запросов контента
- **Лучшая TypeScript поддержка**: Полная типизация для автодополнения и проверки типов
- **Расширяемость**: Возможность создания кастомных парсеров и трансформаторов контента
- **Интеграция с Nuxt 3**: Полная совместимость с экосистемой Nuxt 3 и Nitro сервером

### **Общие преимущества выбранного стека:**

- **Простота обновления контента**: Редакторы могут обновлять информацию через интерфейс GitHub без знания программирования
- **Автоматизация**: После каждого коммита в ветку `main` происходит автоматическая сборка и деплой
- **Производительность**: Nuxt 3 с генерацией статических страниц обеспечивает мгновенную загрузку
- **Безопасность**: Отсутствие базы данных и административной панели снижает поверхность атак
- **Масштабируемость**: Возможность легко добавлять новые страницы и разделы через создание файлов контента

## **1.4. Основные принципы работы с контентом (Git-ориентированный подход)**

В отличие от традиционных CMS (WordPress, Joomla), где контент хранится в базе данных, в данном проекте используется **git-ориентированный подход**:

### **Основные концепции:**

1. **Контент как код**
   ```
   /content
   ├── clinic/
   │   ├── about/index.md
   │   └── history/index.md
   ├── dimensions/
   │   ├── pgysculture/index.md
   │   └── therapy/index.md
   └── ...
   ```

2. **Версионность через Git**
    - Каждое изменение контента фиксируется в истории Git
    - Возможность отслеживать, кто и когда внёс изменения
    - Простой откат к предыдущим версиям при необходимости

3. **Прямое редактирование в GitHub**
    - Редакторы могут обновлять контент через веб-интерфейс GitHub
    - Встроенный редактор с подсветкой синтаксиса для Markdown и YAML

4. **Автоматический деплой по принципу CI/CD**
   ```yaml
   # Пример процесса деплоя
   commit -> push to main -> GitHub Actions -> build -> FTP upload -> live site
   ```

### **Пример рабочего процесса для редактора:**

```markdown
1. Зайти на GitHub в папку `/content/clinic/about/`
2. Найти файл `index.md` и нажать "Edit"
3. Внести изменения в тексте
4. Заполнить поле "Commit changes":
   - Summary: "Обновление информации о клинике"
   - Description: "Добавлены новые достижения клиники"
5. Нажать "Commit changes directly to the `main` branch"
6. Через 3-5 минут изменения появятся на сайте
```

### **Преимущества данного подхода:**

- **Прозрачность**: Все изменения видны в истории Git
- **Надёжность**: Резервные копии создаются автоматически
- **Скорость**: Статические файлы загружаются быстрее динамических страниц
- **Экономия**: Не требуется дорогостоящая CMS или хостинг с поддержкой баз данных
- **Гибкость**: Легко изменить структуру сайта или добавить новые типы контента
