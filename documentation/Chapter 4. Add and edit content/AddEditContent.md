2. Нажмите иконку карандаша (Edit this file)
3. Внесите изменения в тексте
4. Прокрутите вниз, заполните:
    - **Commit message:** "Обновление страницы 'О клинике'"
    - **Optional extended description:** Описание изменений
5. Выберите "Commit directly to the `main` branch"
6. Нажмите "Commit changes"

### **4.1.3. `doctors/index.md` – Наши специалисты**

**Важно:** Этот файл только ссылается на данные из `/content/data/doctors/`. Сами данные врачей хранятся в JSON-файлах.

**Путь основного файла:** `/content/clinic/doctors/index.md`  
**Путь данных врачей:** `/content/data/doctors/`  
**Формат данных:** JSON

**Структура файла врача (`1.rodichkin.json`):**
```json
{
  "name": "Родичкин Павел Васильевич",
  "positionFirst": "Учредитель и главный врач Клиники «Передовые технологии здоровья»",
  "positionSecond": "Главный врач и основатель Клиники, доктор медицинских наук, профессор Родичкин Павел Васильевич",
  "avatar": "/images/doctors/rodichkin.png",
  "descriptionTitle": "Стаж лечебной деятельности по мануальной терапии – более 30 лет.",
  "description": "Родился 25 июня 1966 в г. Одинцово Московской области\nВ 1983 г. окончил школу с золотой медалью\nВ 1989 г. окончил лечебный факультет 1 МОЛГМИ им. И.М.Сеченова\nС 1993 г. занимается мануальной терапией\nВ 1996 г. защитил кандидатскую диссертацию"
}
```

**Как добавить нового врача:**
1. Создайте новый файл в `/content/data/doctors/`:
    - Имя файла: `[номер].[фамилия].json` (например, `5.ivanov.json`)
    - Номер должен быть следующим по порядку
2. Добавьте фото врача в `/public/images/doctors/`:
    - Формат: PNG или JPG
    - Имя файла: `[фамилия].png` (например, `ivanov.png`)
    - Размер: рекомендуется 400x400px
3. Заполните JSON по образцу
4. Для отображения на сайте система автоматически отсортирует врачей по номеру в имени файла

**Редактирование существующего врача:**
1. Перейдите к файлу врача, например: `1.rodichkin.json`
2. Отредактируйте любые поля:
    - `name` - ФИО врача
    - `positionFirst` - Должность (отображается в карточке)
    - `positionSecond` - Расширенная должность (для главной страницы)
    - `description` - Полное ойписание (используйте `\n` для переноса строк)
3. Сохраните изменения

### 4.5.2. **"Вопрос-ответ"** (`/content/data/faq/`)

**Формат:** Каждый вопрос - отдельный JSON файл  
**Структура файла:** `[номер].question.json`

**Пример (`1.question.json`):**
```json
{
   "date": "13.04.2013",
   "name": "Ирина Петровна",
   "question": "Подскажите, пожалуйста, нужно ли при грыже 6 мм в поясничном отделе делать операцию? Боли периодически бывают невыносимыми.",
   "answer": "Добрый день Ирина. Вам следует обратиться для консультации к неврологу-вертебрологу для определения тактики лечения. Грыжа 6 мм может иметь различную иррадиацию (медиальную, латеральную). Грамотное воздействие мануального терапевта вполне реально позволит избежать оперативного вмешательства",
   "doctorName": "Родичкин"
}
```

**Как добавить новый вопрос:**
1. Создайте файл в `/content/data/faq/`: `[номер].question.json`
2. Используйте следующий доступный номер
3. Заполните поля:
    - `question` - текст вопроса
    - `name` - автор вопроса
    - `answer` - развернутый ответ (можно использовать HTML теги)
    - `doctorName` - имя специалиста, отвечающего, должно совпадать с фамилией одного из специалиста из списка

## **4.8. Практические примеры редактирования**

### **Пример 1: Изменение телефона в контактах**

1. Перейдите: `https://github.com/klinika-zdorovya/web/blob/main/content/data/contacts/contacts.yaml`
2. Нажмите кнопку "Редактировать" ("Карандаш")
3. Найдите секцию `phones` и подсекцию `phone`
3. Измените номер телефона:
   ```yaml
   phones:
      - description: "Новый комментарий"
        phone: "+79999999999"
   ```
4. Сохраните изменения

### **Пример 2: Добавление новой новости**

1. Нажмите "Add file" → "Create new file" в папке `/content/news/`
2. Имя файла: `2024-12-01.md`
3. Содержимое:
   ```markdown
   ---
   title: 'Открытие нового кабинета массажа'
   date: '2026-02-01'
   preview: 'Рады сообщить об открытии нового современного кабинета массажа'
   ---
   
   # Новый кабинет массажа
   
   Текст новости...
   ``` 
4. Сохраните файл

### **Пример 3: Обновление прайс-листа**

1. Откройте `pricelist.yaml`
2. Найдите нужную услугу
3. Измените цену:
   ```yaml
   - name: 'Внутривенная капельница'
     minPrice: 2600 # было 1300
     maxPrice: 2600 # было 1300
   ```
4. Сохраните изменения

## **4.9. Проверка изменений**

После сохранения изменений:

1. **Автоматический деплой** начнется через 1-2 минуты
2. **Мониторинг процесса:**
    - Перейдите в "Actions" в репозитории GitHub
    - Посмотрите статус последнего workflow
3. **Проверка на сайте:**
    - Через 3-5 минут обновите страницу сайта
    - Убедитесь, что изменения применились
4. **Если что-то пошло не так:**
    - Проверьте логи в GitHub Actions
    - Убедитесь, что синтаксис YAML/JSON правильный
    - Проверьте, что пути к файлам существуют

## **4.10. Рекомендации по работе с контентом**

1. **Перед сохранением:**
    - Проверьте орфографию и пунктуацию
    - Убедитесь, что синтаксис Markdown/YAML/JSON корректен
    - Проверьте пути к изображениям

2. **Именование файлов:**
    - Используйте только латинские буквы, цифры и дефисы
    - Избегайте пробелов и специальных символов
    - Соблюдайте соглашения проекта (даты для новостей, номера для врачей)

3. **Изображения:**
    - Оптимизируйте размер (рекомендуется до 500KB)
    - Используйте описательные имена файлов
    - Храните в соответствующих папках

4. **Текстовый контент:**
    - Разбивайте длинные тексты на подзаголовки
    - Используйте списки для перечислений
    - Добавляйте изображения для наглядности

5. **Частота обновлений:**
    - Новости: 1-2 раза в неделю
    - Публикации: 1-2 раза в месяц
    - Отзывы: по мере поступления
    - Цены: при изменении (уведомлять пациентов заранее)
